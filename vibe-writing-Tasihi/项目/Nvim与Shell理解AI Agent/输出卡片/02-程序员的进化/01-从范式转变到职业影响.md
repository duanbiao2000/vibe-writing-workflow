## 从范式转变到职业影响

如果AI是"脑的延伸"，如果工具变成了"协作者"，那么"编程"会变成什么？程序员会变成什么？这不再是抽象的技术问题，而是关乎职业生存的问题。
## 编程抽象层的历史演进

编程语言的历史是一部"远离机器、靠近人类"的历史：
```
机器码 → 汇编语言 → 高级语言 → 领域特定语言 → 意图语言？
```
每一层都在抽象掉某些细节，释放程序员去关注更高层次的问题：
- **机器码**：无抽象，直接控制硬件
- **汇编**：抽象掉二进制，用助记符
- **高级语言**：抽象掉CPU细节，用数学逻辑
- **DSL**：抽象掉算法细节，用领域术语
- **意图语言**：抽象掉"怎么做"，只保留"做什么"
## 关键问题浮现

如果抽象掉了"怎么做"，那还是"编程"吗？传统编程的定义是"用精确的语言描述计算过程"。但如果"意图语言"的定义是"用自然语言描述期望结果"，这两者有本质区别。

当"怎么做"被AI接管，程序员的价值体现在哪里？这是整个系列要探讨的核心问题。
