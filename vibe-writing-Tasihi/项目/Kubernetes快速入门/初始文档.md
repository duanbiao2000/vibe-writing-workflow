# Kubernetes快速入门 - 初始文档

## 什么是Kubernetes?

Kubernetes（常简称为K8s）是一个开源的容器编排平台，用于自动化应用程序的部署、扩展和管理。它最初由Google设计，现在由Cloud Native Computing Foundation（CNCF）维护。

简单来说，如果你把应用程序想象成货物，那么Kubernetes就像是一个智能仓库管理系统，它知道如何存储、组织、调度和运输这些货物。

## 为什么需要Kubernetes?

在没有Kubernetes的时代，运维人员需要手动管理服务器上的应用。随着容器技术（特别是Docker）的兴起，虽然部署变得更容易了，但在大规模生产环境中仍然面临挑战：

- 如何协调数百个容器?
- 如何在某个容器崩溃时自动重启?
- 如何根据负载自动扩展应用实例?
- 如何在不同服务器之间均衡资源使用?

这些问题正是Kubernetes要解决的。

## 核心概念

### Pod
Pod是Kubernetes中最小的部署单元。一个Pod可以包含一个或多个紧密相关的容器，这些容器共享存储、网络等资源。

### Node
Node是运行Pod的工作节点，可以是物理机或虚拟机。每个Node都运行着必要的服务来管理Pod。

### Service
Service定义了一组Pod的访问策略，提供稳定的网络端点，使得应用间的通信不受Pod变化的影响。

### Deployment
Deployment负责管理Pod的生命周期，确保指定数量的Pod副本始终在运行。

## 工作原理

Kubernetes集群由两种类型的节点组成：
1. **控制平面节点（Control Plane）**：负责整个集群的管理和决策
2. **工作节点（Worker Node）**：运行实际的应用程序

当你提交一个应用部署请求时，Kubernetes会：
1. 调度Pod到合适的Node上
2. 监控Pod的健康状态
3. 在Pod失效时自动重启
4. 根据配置自动扩展或收缩应用实例

## 快速入门要点

要在20%的精力内掌握80%的内容，重点关注以下几个方面：

1. **基本概念**：理解Pod、Service、Deployment等核心对象
2. **kubectl命令行工具**：学会基本的操作命令
3. **YAML配置文件**：了解如何定义应用部署配置
4. **实践操作**：在本地或云环境中动手部署简单应用

掌握了这些核心内容，你就具备了使用Kubernetes的基本能力，可以进一步深入学习更高级的功能。
