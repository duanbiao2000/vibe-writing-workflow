# 对话论述自动保存机制

**创建时间**：2026-03-13
**版本**：v1.0
**适用范围**：本仓库所有项目

---

## 机制概述

确保每次对话论述内容都能自动保存到项目的`对话论述.md`文件中，避免内容丢失。

---

## 三层保障架构

```
┌─────────────────────────────────────────────────┐
│  第1层：Agent级规范（主要机制）                      │
│  学习Agent每轮对话后自动保存                        │
│  可靠性：★★★★★                                    │
├─────────────────────────────────────────────────┤
│  第2层：Hook兜底（安全保障）                        │
│  SessionEnd hook自动保存未保存内容                  │
│  可靠性：★★★★☆                                    │
├─────────────────────────────────────────────────┤
│  第3层：项目信息追踪（状态提醒）                      │
│  项目信息中记录"对话论述同步状态"字段                  │
│  可靠性：★★★☆☆                                    │
└─────────────────────────────────────────────────┘
```

---

## 第1层：Agent级规范（主要机制）

### 实现位置
`_系统/学习Agent.md` - 步骤5

### 工作流程
1. 每轮对话完成后，AI自动保存论述内容到`对话论述.md`
2. 追加格式遵循统一模板
3. 更新项目信息中的"对话论述同步状态"字段

### 保存格式模板
```markdown
## 第N轮对话：[核心问题摘要]

**核心问题**：
- 问题1
- 问题2

### 论述：[论述主题]

[完整的论述内容，保留原始性和完整性]

---
```

### 关键原则
- **完整性**：保留所有论述内容，不压缩、不提炼
- **原始性**：保持论述的原始表达和结构
- **及时性**：每轮对话完成后立即保存
- **格式一致**：遵循统一的格式模板

---

## 第2层：Hook兜底机制 ✅

### 文件位置
`.claude/hooks/sessionEnd.js`

### 功能实现
SessionEnd hook在对话结束时检查：

1. **查找项目信息文件**：在`项目/*/项目信息.md`中定位当前项目
2. **提取同步状态**：读取"最新同步轮次"和"状态"字段
3. **检查对话论述文件**：定位`对话论述.md`
4. **提取实际轮次**：从对话论述中解析最新的轮次编号
5. **对比差异**：
   - 实际轮次 < 记录轮次 → 警告：可能存在遗漏
   - 实际轮次 > 记录轮次 → 警告：同步状态过时
   - 状态包含"待同步" → 警告：有未保存内容

### 警告示例
```
⚠️ 对话论述可能存在遗漏
   项目信息显示：第6轮已同步
   对话论述实际：第5轮
   差异：1轮内容可能未保存
   对话论述路径：项目/每日写作重塑大脑/对话论述.md
```

### 技术说明
- 使用Node.js原生模块（fs, path）
- 正则表达式解析轮次信息
- 非侵入式检查，不修改文件内容
- 仅在检测到异常时输出警告

## 第3层：项目信息追踪（状态提醒）

### 字段定义
在`项目信息.md`中添加：

```markdown
## 💬 对话论述同步
**最新同步轮次**：第X轮（[YYYY-MM-DD HH:MM]）
**状态**：✅ 已同步 / ⚠️ 待同步
```

### 维护规则
- **更新者**：学习Agent
- **更新时机**：每轮对话保存到`对话论述.md`后
- **作用**：追踪对话论述是否已同步，便于发现遗漏

---

## 验证检查清单

每次新项目启动时，验证以下项目：

- [ ] 项目信息.md包含"对话论述同步"字段
- [ ] 对话论述.md文件已创建（包含初始头部信息）
- [ ] 学习Agent已加载步骤5（保存对话论述）
- [ ] 检查清单中包含对话论述保存检查项

---

## 故障排查

### 问题：对话论述未同步

**检查步骤**：
1. 读取项目信息.md，查看"对话论述同步"状态
2. 读取对话论述.md，确认最新轮次
3. 对比当前轮次，找出缺失的轮次
4. 手动从对话历史中恢复缺失内容

### 问题：格式不一致

**解决方案**：
- 参考本文档的"保存格式模板"
- 确保每轮对话包含：核心问题、论述内容、分隔线
